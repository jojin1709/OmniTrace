Overview

One input in, a correlated intelligence dossier out. OmniTrace classifies your input, fans out across specialised modules, merges everything into an identity graph, and renders a report in the format you choose.

ℹ️
Works instantly. OmniTrace ships with realistic demo data, so every command runs with zero setup. Add API keys in config/keys.yaml to switch modules to live results.
🎯

Any Input

Name, username, email, phone, or photo — even a combination for stronger correlation.

🧩

15+ Modules

Social, breach, face, phone, email, crypto, dark web, network, documents and more.

🕸️

Identity Graph

All findings are linked into a relationship graph with per-finding confidence scores.

📄

4 Report Formats

Console, JSON, interactive HTML, or PDF dossiers — pick what fits your workflow.

🌐

CLI + REST API

Use it from the terminal, or run the Flask server with a web dashboard.

🔑

Live or Demo

Plug in API keys for real data; otherwise everything runs on built-in mock data.

Installation

Requires Python 3.9+. All commands work immediately with demo data — no keys needed.

bash
# 1. Clone the repository
git clone https://github.com/jojin1709/OmniTrace.git
cd OmniTrace

# 2. Install dependencies
pip install -r requirements.txt

# 3. (Optional) Add API keys for live data
cp config/keys.example.yaml config/keys.yaml
# ...then edit config/keys.yaml

CLI Commands

Every command below works out of the box. Add --help to any command for its full options.

🔍 Identity Trace

bash
# Trace by a single input
python cli.py trace --name "John Doe"
python cli.py trace --username johndoe
python cli.py trace --email john@example.com
python cli.py trace --phone +1234567890
python cli.py trace --photo ./photo.jpg

# Multiple inputs = best correlation
python cli.py trace --name "John Doe" --username johndoe --email john@example.com

# Scan depth + relationship graph
python cli.py trace --name "John Doe" --mode deep --graph

📄 Output Formats

bash
python cli.py trace --name "John Doe" --output console
python cli.py trace --name "John Doe" --output json
python cli.py trace --name "John Doe" --output html --file dossier.html
python cli.py trace --name "John Doe" --output pdf

👤 Username · ✉️ Email · 📞 Phone

bash
python cli.py check-username --username johndoe
python cli.py check-username --usernames-file usernames.txt

python cli.py investigate-email --email john@example.com --breaches --social --gravatar

python cli.py investigate-phone --phone +1234567890 --carrier --social --reputation

🛡️ Breach · 🌍 Social · 📍 Place

bash
python cli.py search-breaches --email john@example.com
python cli.py search-breaches --password "password123"   # reuse detection

python cli.py search-social --name "John Doe" --platform linkedin

python cli.py identify-place --photo ./building.jpg --details --map

🕶️ Dark Web · 🪙 Crypto · 📑 Documents

bash
python cli.py search-darkweb --name "John Doe"
python cli.py monitor-darkweb --email john@example.com --continuous --interval 24h

python cli.py trace-crypto --btc 1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa
python cli.py trace-crypto --eth 0x742d35Cc6634C0532925a3b844Bc9e7595f2bD18

python cli.py analyze-document --file document.pdf
python cli.py analyze-document --file image.png --stego

🌐 Network Intelligence

bash
python cli.py whois --domain example.com
python cli.py dns-enum --domain example.com
python cli.py ip-reputation --ip 1.2.3.4

REST API & Dashboard

Run the Flask server to expose a JSON API and an interactive web dashboard.

bash
# Start the server (either works)
python api.py
python web/dashboard.py

# → open http://localhost:5000/dashboard

# JSON trace via curl
curl -X POST http://localhost:5000/trace \
  -H "Content-Type: application/json" \
  -d '{"name": "John Doe", "email": "john@example.com"}'

/identify-placePOSTPlace identification
/investigate-emailPOSTEmail investigation

Live Data / API Keys

Copy the template and add keys for any services you have. Modules use live data automatically when their key is enabled, and fall back to demo data otherwise.

config/keys.yaml
services:
  haveibeenpwned:
    enabled: true
    api_key: "YOUR_KEY"
  clearbit:
    enabled: true
    api_key: "YOUR_KEY"
🔒
Your secrets stay local. config/keys.yaml is git-ignored, so keys are never committed to the repository.

How The Engine Works

A four-stage pipeline turns raw input into a correlated dossier.

① Input ProcessorClassifies the input type, extracts leads, builds the target profile
↓
② Module ExecutionRuns every applicable module for the given input(s)
faceimagenamesocialusernameemailphonebreachdarkwebcryptolocationnetworkdocument
↓
③ Correlation EngineBuilds the identity graph and computes confidence scores
↓
④ Report GeneratorRenders a Console / JSON / HTML / PDF dossier

Confidence Scoring

Every finding carries a confidence score based on the strength and independence of its evidence.

ScoreMeaning
0.9 – 1.0Certain — verified by multiple independent sources
0.7 – 0.9High confidence — strong evidence from 2+ sources
0.4 – 0.7Medium confidence — single source or partial match
0.1 – 0.4Low confidence — weak match, possible false positive
0.0No match found
