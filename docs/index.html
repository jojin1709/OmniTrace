Overview

One input in, a full dossier out

OmniTrace classifies your input, fans out across specialised modules, merges everything into an identity graph, and renders a report in the format you choose.

ℹ️
Works instantly. OmniTrace ships with realistic demo data, so every command runs with zero setup. Add API keys in config/keys.yaml to switch modules to live results.
🎯

Any Input

Name, username, email, phone, or photo — or a combination for stronger correlation.

🧩

15+ Modules

Social, breach, face, phone, email, crypto, dark web, network, documents and more.

🕸️

Identity Graph

Findings are linked into a relationship graph with per-finding confidence scores.

📄

4 Report Formats

Console, JSON, interactive HTML, or PDF dossiers — pick what fits your workflow.

🌐

CLI + REST API

Use it from the terminal, or run the Flask server with a web dashboard.

🔑

Live or Demo

Plug in API keys for real data; otherwise everything runs on built-in mock data.

Setup

Installation

Requires Python 3.9+. All commands work immediately with demo data — no keys needed.

bash
# 1. Clone the repository
git clone https://github.com/jojin1709/OmniTrace.git
cd OmniTrace

# 2. Install dependencies
pip install -r requirements.txt

# 3. (Optional) Add API keys for live data
cp config/keys.example.yaml config/keys.yaml

Server

REST API & Dashboard

Run the Flask server to expose a JSON API and an interactive web dashboard.

bash
# Start the server (either works)
python api.py
python web/dashboard.py     # → http://localhost:5000/dashboard

# JSON trace via curl
curl -X POST http://localhost:5000/trace \
  -H "Content-Type: application/json" \
  -d '{"name": "John Doe", "email": "john@example.com"}'

Configuration

Live Data / API Keys

Add keys for any services you have. Modules use live data automatically when their key is enabled, and fall back to demo data otherwise.

config/keys.yaml
services:
  haveibeenpwned:
    enabled: true
    api_key: "YOUR_KEY"
  clearbit:
    enabled: true
    api_key: "YOUR_KEY"
🔒
Your secrets stay local. config/keys.yaml is git-ignored, so keys are never committed to the repository.
Internals

How The Engine Works

A four-stage pipeline turns raw input into a correlated dossier.

1Input ProcessorClassifies the input type, extracts leads, builds the target profile
↓
2Module ExecutionRuns every applicable module for the given input(s)
faceimagenamesocialusernameemailphonebreachdarkwebcryptolocationnetworkdocument
↓
3Correlation EngineBuilds the identity graph and computes confidence scores
↓
4Report GeneratorRenders a Console / JSON / HTML / PDF dossier
Scoring

Confidence Scoring

Every finding carries a score based on the strength and independence of its evidence.

ScoreMeaning
0.9 – 1.0Certain — verified by multiple independent sources
0.7 – 0.9High confidence — strong evidence from 2+ sources
0.4 – 0.7Medium confidence — single source or partial match
0.1 – 0.4Low confidence — weak match, possible false positive
0.0No match found
